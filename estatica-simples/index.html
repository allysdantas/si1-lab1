Sistemas da Informação I

Informações gerais

O objetivo da disciplina é discutir técnicas de projeto de software. Haverá foco particular em técnicas utilizadas para melhorar o reuso e a flexibilidade de software desenvolvido usando Orientação a Objeto, mas discutiremos também projeto arquitetural.

Sobre o curso

Conteúdo

Fundamentos; princípios de design (modularidade, coesão, acoplamento); paradigmas de design (funcional, orientado a objetos, outros); projeto orientado a objetos; linguagem de modelagem; abstração, herança, interfaces; bibliotecas e frameworks; padrões de design; noções de arquitetura de software.

Objetivos

No final do curso, o aluno deve ser capaz de:

  1. Descrever os objetivos do projeto de software e seu papel no desenvolvimento de um software
  2. Definir o que são padrões de projeto e nomear um conjunto de padrões
  3. Criticar a necessidade e/ou oportunidade da aplicação de padrões em um contexto
  4. Descrever e criticar uma arquitetura de software
  5. Aplicar boas práticas e padrões de projeto na transformação de uma especificação em um projeto flexível e reutilizável Produzir um software complexo utilizando boas práticas e padrões de projeto
  6. Produzir um software web que utilize tecnologias atualmente populares no mercado
  7. Conectar o software sendo implementado às necessidades do cliente que o demanda
  8. Criar software multithreaded
